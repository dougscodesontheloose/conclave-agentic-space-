# Visual Identity System

> This file is the canonical visual identity reference for all Conclave agents.
> When creating visual content, agents MUST consult this file before making design decisions.
> It exposes design branches that are selected based on content type matching.

<!--
  INSTRUCTIONS FOR AI AGENTS — IDENTITY BUILDER PROTOCOL:

  This file should be built through a structured discovery process.
  If this file is empty or contains only these instructions, trigger the Visual Identity Builder:

  ## Step 1: Analyze Existing Assets
  If the user has a website, social media, or existing brand materials:
  - Use WebFetch to analyze their website's visual language (colors, fonts, spacing)
  - Extract the implicit design rules from their best content
  - Present findings and ask for confirmation

  ## Step 2: Reference Collection (Optional)
  If the user has a reference folder (e.g., `ref_visual_style/`):
  - Analyze the reference images for patterns
  - Identify recurring themes, color temperatures, layout preferences
  - Synthesize into design principles

  ## Step 3: Build the Identity Document
  Structure the findings into the sections below:
  - Core philosophy (the "vibe")
  - Layout patterns
  - Color palette (with hex codes)
  - Typography system
  - Signature elements
  - Content-type style matching (the "style dictionary")

  ## Step 4: Create the Style Dictionary
  The Style Dictionary is a mapping system that matches content types to visual styles.
  Before generating any visual asset, the design agent should:
  1. Analyze the content's topic and tone
  2. Match it against the style dictionary entries below
  3. Select the appropriate visual package
  
  This ensures visual consistency while allowing variety across different content types.
-->
