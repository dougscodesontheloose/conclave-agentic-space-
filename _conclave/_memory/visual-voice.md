# Visual Voice Guide

> This file defines your personal visual design language — the "voice" your brand speaks through design.
> It is consumed by all visual agents (designers, renderers, carousel builders) in every squad.

<!--
  INSTRUCTIONS FOR AI AGENTS — VISUAL VOICE DISCOVERY PROTOCOL:
  
  This file should be populated through an interactive discovery session with the user.
  Do NOT fill this with generic defaults. Instead, run the following interview:

  ## Phase 1: Visual Taste Discovery
  Ask the user to describe (or share reference images/URLs) for:
  
  1. "Show me 3 designs you love — websites, posters, apps, architecture, anything."
  2. "What visual style makes you feel 'this is premium'?"
  3. "What visual style makes you cringe or feels generic?"
  4. "If your brand were a physical space, what would it look like?"

  ## Phase 2: Structural Preferences
  5. "Do you prefer dense, information-rich layouts or breathing, minimal ones?"
  6. "How do you feel about visible grids, borders, and structural elements?"
  7. "Symmetry or asymmetry?"
  8. "Sharp corners or rounded? Glass effects or solid?"

  ## Phase 3: Color System
  9. "What colors represent your brand? (Provide hex codes if you have them)"
  10. "What colors should NEVER appear in your outputs?"
  11. "Do you prefer dark mode, light mode, or contextual switching?"

  ## Phase 4: Typography
  12. "What fonts do you like? Do you have brand fonts already?"
  13. "Do you prefer sans-serif, serif, or a mix?"
  14. "Should code/technical content use monospaced fonts?"

  ## Phase 5: Design Packages (Optional)
  After gathering answers, propose 2-3 "design packages" — named visual profiles the user can
  switch between depending on content type. Example:
  - Package A: "Technical Precision" — for tutorials, data, tech content
  - Package B: "Warm Editorial" — for thought leadership, reflections, stories
  - Package C: "Bold Impact" — for announcements, hot takes, urgent content

  Save all findings below in structured sections.
